Multi-Agent Story Generation Process

To ensure high-quality and structurally sound stories, we use a multi-agent workflow powered by LLMs and programmatic validation.

Architecture

[User Input] | v Creator Agent (LLM) <--- (Feedback Loop) ---+ | | v | [Story JSON] | | | v | Structural Validation (Pydantic) ----------+ | | v | Validator Agent (LLM) ----------------------+ | v [Final Validated Story]

Step-by-Step

  1. Creator Agent: Generates the initial story based on the user's topic, CEFR level, and age range. It follows specialized storytelling skills.
  2. Structural Validation: A Pydantic model checks the JSON structure, ensuring all referenced nodes exist and the format is correct.
  3. Validator Agent: A second LLM pass reviews the story for narrative quality, age-appropriateness, and logical consistency. It also receives any errors from the structural validation.
  4. Feedback Loop: If either validation step fails, the feedback is sent back to the Creator Agent to fix the specific issues. This repeats up to 3 times.
